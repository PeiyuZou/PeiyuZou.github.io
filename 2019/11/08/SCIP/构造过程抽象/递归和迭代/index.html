递归和迭代

 

计算过程的“形状”

从线性递归举例子开始，这里使用Lisp语言描述6的阶乘（6!）：

(define (factorial n)
    (if (= n 1)
        1
        (* n (factorial (- n 1)))))

另外一种计算的方式，是利用计数器，先乘起1和2，结果再乘以3，以此类推，直到n：

(define (factorial n)
    (fact-iter 1 1 n))

(define (fact-iter product counter max-count)
    (if (> counter max-count)
        product
        (fact-iter (* counter product)
            (+ counter 1)
            max-count)))

同样用代换模型，得到计算过程：

前者是一个推迟执行的过程，它是先展开后收缩的形状，仅在“收缩”阶段执行实际的计算。前者称为递归，递归计算需要解释器维护好“展开”的轨迹，图中的曲线表示实时保存的信息量，这要求的空间复杂度为O(n);
后者在计算过程中，需要保存的信息数量固定。这类仅用固定数量的状态变量描述的嵌套计算过程称为迭代，要求的空间复杂度仅为常数（O(1)）;

尾递归

迭代相较于递归有着明显的优势，但递归仍有适用的场景：比如当我们无法确定执行的规模n时。
一般递归的劣势在于随着问题规模n的扩大，需求的空间成线性增长。使用尾递归，可以将内存开销维持在常量的水平。

return x + tailrec(x + 1);

//改为

return tailrec(x + 1);

这样解释器不必存储每一次嵌套的x变量。
尾递归优化主要是对栈内存空间的优化，这个优化是O(n)到O(1)的；至于时间的优化，其实是由于对空间的优化导致内存分配的工作减少所产生的，是一个常数优化，不会带来质的变化。
