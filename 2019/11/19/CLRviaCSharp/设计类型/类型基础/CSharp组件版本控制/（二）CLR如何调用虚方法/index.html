（二）CLR如何调用虚方法

方法唯一性

所有方法都包含名称、签名和返回类型。包括C#在内的大多数语言只以方法名和参数为准，来确定方法的唯一性，也就是说同名、同参数的方法，就算返回值类型不一样，编译也是通不过的。只有IL汇编语言，才可以通过返回值类型来区分。

方法的分类

方法：

  • 静态方法
  • 实例方法
    • 虚实例方法
    • 非虚实例方法

在类型上执行操作的方法，称为静态方法；在类型的实例上执行操作的，称为实例方法（非静态方法）；实例方法是参与到继承层次结构中的成员，因此又可分为虚实例方法（虚方法）和非虚实例方法。

方法调用指令（call && callvirt）

internal class Employee
{
    //非虚实例方法
    public Int32 GetYearsEmployed() { ... }
    //虚方法
    public virtual String GetProgressReport() { ... }
    //静态方法
    public static Employee Lookup(String name) { ... }
}

编译上面代码时，编译器在程序集的方法定义表中写入3个记录项，每个记录项用一组标志（flag）指明该方法是哪种方法（静态、虚、非虚实例）。标志（flag）是编译器判断如何正确生成IL代码的依据。

CLR提供的两个方法调用指令：

  • call：可用于静态、实例、虚方法。
    • 调用静态方法，必须指定定义方法的类型。
    • 调用实例方法，必须指定引用了对象的变量。该变量本身的类型指明了定义方法的类型。如果变量的类型没有定义此方法，则检查基类型。
    • call指令是在编译时检查的（静态绑定），因此指定的变量为字面调用的变量（编译时类型），并非线程栈当前栈顶的发出调用的变量。由于是静态绑定的，无法检查变量是否为null，都假定不为null，因此call指令可能会引发NullReferenceException。
  • callvirt：可用于实例方法和虚方法，不能用于静态方法。
    • 调用非虚实例方法，变量的类型指明了方法的定义类型
    • 调用虚实例方法，CLR调查发出调用的对象的实际类型，然后以多态方式调用方法。
    • callvirt是后期绑定的（在运行时检查类型），注意调用非虚和虚实例方法的区别，前者是变量的编译时类型用作检查的类型（同call），后者是发出调用的对象的实际类型（也就是当前线程栈上栈顶发出调用的变量指向的对象），本文后面会讨论这一重要的区别。另外，不管虚或非虚，callvirt都会执行变量的null检查，因此执行速度会稍慢于call指令（太小以至于可以忽略）。

无论用call还是callvirt调用实例方法或者虚方法，这些方法通常接受隐藏的this实参作为方法第一个参数，引用要操作的对象。

下面我们对3种方法分别调用，并反汇编出IL代码查看C#编译器是如何工作的：

class Program
{
    static void Main(string[] args)
    {
        Console.WriteLine();   //调用静态方法

        Object o = new Object();
        o.GetHashCode();       //调用虚实例方法
        o.GetType();           //调用非虚实例方法
    }
}

反汇编如下：

可以看到C#编译对静态方法、虚实例方法、非虚实例方法分别使用了call、callvirt、callvirt。前两者都在预料之中。唯独最后一次调用，以callvirt调用了非虚实例方法。
这虽然古怪，但是是可行的。对代码执行JIT编译时，CLR知道GetType不是虚方法，在JIT编译完成后，会非虚地调用GetType。至于为什么不直接生成call来调用非虚实例方法，是因为C#团队觉得需要对发出调用的对象执行null检查。

但是在某些情况下，编译器不得不选择使用call而不是callvirt：

internal class SomeClass
{
    public override String ToString()
    {
        ...
        return base.ToString();
    }
}

这里调用虚方法base.ToString()时，C#编译器生成的是call指令。如果使用callvirt指令，编译器会查找实际发出调用的对象，base是SomeClass类型的成员，用来代指基类型，但是调用时不会入栈；所以发出调用的实际类型是SomeClass，现在会在SomeClass的类型对象的方法表中查找ToString方法并调用，没错，这样会导致递归执行，直到栈溢出。因此这里C#编译器会自动处理为使用call指令调用。

调用值类型的方法

C#编译器调用值类型的方法时使用call指令。原因有下：

  • 值类型密封（sealed），即使值类型含有虚方法，也不用考虑多态性，这样调用会更快。
  • 值类型永不为null，无需null检查
  • 如果以虚方式调用值类型中的虚方法，CLR要获取对值类型的类型对象的引用（因为要使用类型对象中的方法表），这会造成值类型装箱，性能受损。

设计类型时关于虚方法的注意事项

尽量减少虚方法，原因有下：

  • 调用虚方法速度比非虚方法慢
  • JIT编译器不能内嵌虚方法，进一步影响性能
  • 使组件版本控制变得更脆弱（一旦某非密封类发生改变，新类必须保持虚方法调用顺序，以避免中断派生）

 
在定义基类型时，如果需要一个方法有许多多态版本，重载出多个方法，那么应该将最复杂（参数最多）的那个定义为虚方法，其他的为非虚方法，并且在内部可以调用虚版本的那个方法。这样可以改善版本控制，也不影响派生类。
