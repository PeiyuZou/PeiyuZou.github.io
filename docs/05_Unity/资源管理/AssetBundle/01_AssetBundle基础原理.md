## 一、什么是AssetBundle

### 1.1 定义

**AssetBundle（简称AB）** 是Unity提供的一种资源打包格式，本质上是一个**压缩的资源文件容器**。

### 1.2 核心特点

| 特点 | 说明 |
|------|------|
| 归档文件 | 包含特定平台格式的非代码资源（模型、纹理、预制体、音频等） |
| 动态加载 | 可以在运行时动态加载，无需打包在应用内 |
| 热更新支持 | 支持资源的热更新，不需要重新发布整个应用 |
| 减少包体 | 减少初始安装包大小 |
| 平台相关 | 每个平台需要单独打包（Windows、Android、iOS等） |

### 1.3 使用场景

✅ **适合使用AB的场景：**

- 大量资源需要分包加载
- 需要热更新游戏内容
- DLC和扩展包
- 减少初始安装包体积
- 按需下载资源

❌ **不适合使用AB的场景：**

- 游戏必需的核心资源（应该打包在应用内）
- 频繁变化的小资源（管理成本高）
- 代码热更新（AB只能更新资源，不能更新C#代码）

## 二、AssetBundle的结构

当我们用 AssetStudio 等工具解包一个 AB 包后，会得到下面这样的若干文件：

![AssetBundle解包结果](asset_bundle_01.png)

不带后缀的这个是 AssetBundle 主文件，而另外一个是资源流文件（Resource Stream），资源流文件有时候后缀是 `*.resource`，它是旧格式，但是原理一致。

要搞清楚它们都是什么，需要结合官方给的示意图：

![官方的 AssetBundle 结构示意图](asset_bundle_02.png)
/// caption
官方的 AssetBundle 结构示意图
///

对于序列化文件而言，它们始终存储在主文件中；而资源文件，由于二进制数据块较大（比如 Texture 像素数据），被单独打进资源流文件中。之所以这样拆分是为了降低内存和 IO 压力，加载 AssetBundle 时无需将整个 AB 包 IO 起来并全部读入内存，只有在必须使用其中的数据时 IO 并加载。

### 实验

图集文件和音频文件，大部分都是资源文件数据，所以它们解包后，资源流文件大小大于主文件：
