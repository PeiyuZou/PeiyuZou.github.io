C# 调用 Lua 还有一个重要的话题，那就是复杂参数怎么传递。之前的例子中都是以基本类型（Primitive）为例子，因为它们的处理足够简单（和 lua 的基本类型一一对应，提前注册好了压栈函数，直接 P/Invoke 调用入栈），不会影响我们分析 C# 调用 Lua 的流程。

那么如果参数是一个复杂对象，比如 C# 的一个对象，这个对象是如何入栈的呢？Lua 又是如何访问并使用它的呢？我们下面会以 `PushByType<GameObject>(L, go)` 为例子来分析。

!!! important "先说结论"
    C# 对象传递给 Lua，并不是直接把整个 GameObject 直接传过去，和之前 C# 获取 lua function 一样，因为两边 GC 不同，不能直接传递原始对象。

    C# 对象被登记到 translator 的对象池中，仍然由 C# 侧维护。对象池负责返回一个索引号 index，这个 index 作为 Lua 访问这个对象的凭证。实际压栈的，是 Lua 侧创建的一个 userdata 句柄，它持有这个 index 以便后续访问真实的 C# 对象。

    但光有 index 是不够的，userdata 还需要知道有哪些成员可以访问，以 GameObject 类型为例，当第一次压栈一个 GameObject 类型的参数时，translator 会调用 `UnityEngine_GameObjectWrap` 桥接类的 `__Register(L)` 函数给 Lua 注册一个描述 GameObject 类型的 metatable，其中包含一些成员函数、属性、getter/setter 等，对应的函数指针也设置好了，Lua 一侧通过 P/Invoke 可以直接调用 C# 对象的函数。最后 userdata 通过 type_id 绑定这个 metatable，这样就可以直接作为真实的 C# 对象来使用。但是要注意，这个 userdata 本质上是一个句柄，并非真实 C# 对象。


## 心智模型建立

1. Lua 里拿到的是不是整个 GameObject 或者它的副本？

不是。Lua 拿到的是一个 userdata 句柄，背后仍指向原始 C# GameObject 实例。

2. 为什么 Lua 能写 go.transform、go:SetActive(true)？

因为这个 userdata 绑定了 GameObject 的 metatable，元表里提前注册了 getter / setter / method 包装函数。

3. 同一个 GameObject 多次 Push 会重复创建 userdata 吗？

通常不会。xLua 会优先查 reverseMap 和弱表缓存，命中时直接复用旧 userdata。

4. 首次 Push 为什么更慢？

因为首次需要把 UnityEngine.GameObject 的 metatable 延迟注册出来。

### 3. Push 先检查是否可以复用之前的 userdata

```cs title=""
public void Push(RealStatePtr L, object o)
{
    if (o == null)
    {
        LuaAPI.lua_pushnil(L); // null 直接压栈一个 nil
        return;
    }

    int index = -1;
    Type type = o.GetType();
#if !UNITY_WSA || UNITY_EDITOR
    bool is_enum = type.IsEnum;
    bool is_valuetype = type.IsValueType;
#else
    bool is_enum = type.GetTypeInfo().IsEnum;
    bool is_valuetype = type.GetTypeInfo().IsValueType;
#endif
    bool needcache = !is_valuetype || is_enum;

    // GameObject 对象会尝试从 reverseMap 取缓存
    if (needcache && (is_enum ?
        enumMap.TryGetValue(o, out index) :
        reverseMap.TryGetValue(o, out index)))
    {
        // 不管 C# reverseMap 有没有，都要看看 userdata 还在不在
        if (LuaAPI.xlua_tryget_cachedud(L, index, cacheRef) == 1)
        {
            return;
        }
    }
    ...
    // 剩余步骤：
    // 1.该类型首次入栈，构建元表、获取 type_id
    // 2.对象入池，分配 index
    // 3.构建 userdata 并压入栈
}
```

查 reverseMap 是为了拿之前存过的旧 index，但就算 C# 这边缓存还在，也要尝试取下 lua userdata 看看是否还在，因为可能已经被回收了。这里 cacheRef 是登记在 lua REGISTER 全局表中的一个弱表，用来存 userdata 的。这里如果 index 在，userdata 也在的话，直接复用这个 userdata 就行，已经在栈上了，直接 return 。

### 5. 延迟注册元表

```cs title=""
public bool TryDelayWrapLoader(RealStatePtr L, Type type)
{
    if (loaded_types.ContainsKey(type)) return true;
    loaded_types.Add(type, true);

    ...
}
```

### 6. xxxWrap.__Register 构建真正的元表

```cs title=""

    Utils.RegisterFunc(L, Utils.SETTER_IDX, "layer", _s_set_layer);
```

这里给元表注册实例方法、getter/setter、类静态方法等，对应的函数指针直接被包含进去，实际发生调用时通过 P/Invoke 调回来。

### 7. 对象入池，分配 index

到这里 GameObject 对应的元表已经就绪了，接着就是 C# 对象加入缓存对象池，并且分配一个 index 作为引用交给 lua 使用。

```cs title=""
int index = addObject(o, is_valuetype, is_enum);

int addObject(object obj, bool is_valuetype, bool is_enum)
{
    int index = objects.Add(obj); // 对象池负责返回一个 index
    if (is_enum)
    {
        enumMap[obj] = index;
    }
    else if (!is_valuetype)
    {
        reverseMap[obj] = index;   // GameObject 走这里
    }
    return index;
}
```

这里看的出来是双向缓存：

- `objects[index] = go`：正向池，后续 Lua 通过 index 能找回 C# 实例
- `reverseMap[go] = index`：反向池，后续再 push 同一个 go 能快速命中缓存

### 8. 构建 userdata 并压入栈

```cs title=""
LuaAPI.xlua_pushcsobj(L, index, type_id, needcache, cacheRef);
```

参数：

- index：这个 userdata 代表 objects 池里的第几个 C# 对象
- type_id：这个 userdata 应该挂哪张 metatable（这里是 GameObject 的）
- needcache = true：这个 userdata 需要放进缓存弱表，便于以后复用
- cacheRef：缓存弱表在 registry 里的引用号

到这里，GameObject 对象入栈就完成了，栈上的是一个 userdata 句柄，lua 使用它来操作真正的 GameObject 对象。

## Lua 通过 userdata 调用 C# 对象

现在反过来看 lua 使用这个 C# 对象：

```lua title=""
function FooFunction(go)
    ...
    go:SetEnable(true)
    ...
end
```

这里调用 `SetEnable` 函数将触发 go 元表的对应键，最终调用到 `UnityEngineGameObjectWrap._m_SetEnable`：

```cs title=""
[MonoPInvokeCallbackAttribute(typeof(LuaCSFunction))]
static int _m_SetEnable(RealStatePtr L)
{
    try
    {
        ObjectTranslator translator = ObjectTranslatorPool.Instance.Find(L);

        UnityEngine.GameObject gen_to_be_invoked =
            (UnityEngine.GameObject)translator.FastGetCSObj(L, 1);
        {
            System.Type _type = (System.Type)translator.GetObject(L, 2, typeof(System.Type));

            gen_to_be_invoked.SetEnable(_type);

            return 0;
        }
    }
    catch(System.Exception gen_e)
    {
        return LuaAPI.luaL_error(L, "c# exception:" + gen_e);
    }
}
```

`FastGetCSObj` 从 userdata 里取出 index，再去 `objects` 池里找回原始 GameObject，`gen_to_be_invoked.SetEnable(_type);` 已经是真正的 C# 对象在执行调用了。
