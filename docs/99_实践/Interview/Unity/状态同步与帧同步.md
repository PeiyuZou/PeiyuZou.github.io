# 帧同步 vs 状态同步及确定性

## 题目

帧同步与状态同步的核心区别、优缺点与各自适用的游戏类型？帧同步如何保证「确定性」（deterministic）？浮点数、物理、随机数这些坑怎么处理？（非高频但有技术含量，MOBA/对战/MMO 岗高频）

## 参考答案

### 帧同步（Lockstep）

客户端只同步**操作输入**，各客户端本地跑相同逻辑，得到相同结果；服务器转发/校验输入。

- 优点：流量小（只传输入）；天然支持回放（记录输入即可）；服务器计算压力小。
- 缺点：**必须保证确定性**，任何不确定都会蝴蝶效应式不同步；客户端性能要求一致；断线重连难（需追赶或快照）。
- 适用：RTS、格斗、MOBA、回合制等需要精确一致模拟且单位多的类型。

### 状态同步（State Sync）

服务器跑权威逻辑，把状态（位置、血量等）广播给客户端，客户端插值显示。

- 优点：不需要确定性；断线重连容易（服务器下发当前状态）；服务器权威便于反外挂。
- 缺点：流量大；服务器计算压力大；回放需服务器录制状态。
- 适用：MMO、FPS、多数服务器权威的多人游戏。

### 帧同步如何保证确定性

1. **浮点数**：禁用 `float`/`double`，改用**定点数**（Q 格式整数）运算——不同 CPU/编译器/FMA 下浮点结果可能不同。
2. **物理引擎**：不用 Unity PhysX（不确定），用自研定点物理或确定性物理库。
3. **随机数**：使用确定性 PRNG（如 LCG），所有客户端同种子，且调用顺序一致。
4. **集合遍历顺序**：避免依赖哈希遍历顺序的 `Dictionary` 遍历（顺序不确定），排序需稳定且比较函数确定。
5. **逻辑/表现分离**：逻辑帧固定步长（如 66fps 逻辑帧），表现层插值；逻辑层禁止任何 `Time.deltaTime` 之类的不确定量。
6. **时钟与帧号同步**：服务器统一帧号，客户端按帧号追赶，丢包则等待/补帧。
7. **反外挂校验**：服务器校验关键操作或对状态做哈希校验，发现不一致踢出。

> 答题亮点：点出「浮点 / 物理 / 随机 / 集合顺序」四大确定性杀手，并强调逻辑帧固定步长 + 表现层插值的经典架构。
